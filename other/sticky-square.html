Inline template: sticky square

"Sticky square": this is what we call a template which shows up at a corner of a page and remains there despite scrolling, as if it's glued. And usually it's squarish as it appears to be the most convenient shape to fit all content we need.

Technically, it's a typical inline template, simply put into another context: into a fixed-positioned container instead of a one among paragraphs. This means that on Piano end you need a simple template while on your end we would ask you to add a few lines of CSS code.

Setting up a template

There're 2 ways to set up an inline template: on your end or in Composer. More information on this you will find in our documentation, in the links just mentioned. Please pay attention to an inline type of templates.

Make the template sticky

Let's say your container for an inline template has a CSS class .piano-container. The template will render there taking 100% of its width. So you need to specify container's width along with position on a page. Here is a code example which you may add to your CSS, probably after some tweaking:

    .piano-container {
      position: fixed;
      right: 40px;
      bottom: 40px;
      width: 350px;
    }

You may make it emerge smoothly, as if slides in. Then just add some animation:

    .piano-container {
      position: fixed;
      right: 40px;
      bottom: 40px;
      width: 350px;
      transform: translateX(500px);
      animation-name: slideLeft;
      animation-duration: 1s;
      animation-timing-function: cubic-bezier(0.05, 0.36, 0.29, 1.1);
      animation-fill-mode: forwards;
    }

    @keyframes slideLeft {
      from {
        transform: translateX(500px);
      }
      to {
        transform: translateX(0);
      }
    }

Animation parameters such as duration or timing function may be adjusted. You can also make the widget slide up or roll from another corner or fly from above. Anything you want. CSS animations at your service.

Specifying a template size

Piano inline templates simply take 100% of a parent container width. By the parent container we mean that <div> you specify on your end for Piano template to show up. Regarding its height, the browser counts it automatically according to the template contents. So nothing needs to be defined. Oppositely, please, let Piano template be parsed freely — things like setting up a fixed height would prevent it from displaying correctly.
