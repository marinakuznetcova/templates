Modals: multistep

A multistep templates consist of two or more screens. Usually a customer navigates between them using buttons or links.

The most common example of a multistep template is adblocker intercept where on the first screen a user chooses either to subscribe or to turn the adblocker extension off. In the latter case on the second screen there're different extensions to choose. And on the third screen there's an instruction on how to turn the selected adblocker off.

Another example is to show a list of main 3 offers on the first screen and additional options on the second.

Turning a regular modal to multistep

When considering implementation of a multistep modal one could think of using several separate modal templates. Although we recommend to include all screens into one template and handle them using simple Javscript. It would work faster with no delay between loading different modals, and also it's easier to implement. Basically all we need to do is to show or hide screens on click (or any other event).

An example of code can be borrowed from the demo template code.

Setting up a modal

There're 2 ways to set up a modal: on your end or in Composer. More information on this you will find in our documentation, in the links just mentioned. Please pay attention to a modal type of templates.

Specifying a modal size

Modal width is being set in a special tag <config> which should be added to the very top of a template code. Specifically, you'd want to add a line like this
<config width="600"></config>
where "600" is a modal width, measured in pixels. You can set any amount here. Please keep in mind that for more narrow screens than the specified width (here in example for screens which are 600px wide and less) the template will take 100% of the screen. So, for example, on a regular smartphone with a screen width 375px the template width would also be 375px.

Customizing a close button

There're 2 options of close buttons in modals: Piano default close button and a custom one. Default close button is overflowing modal borders, and basically is located on your end. All customization should also take place on your end, here is more info on how to do this.

Custom close button might be added right to a template code. It can also be a minimalist X or a colorful button saying something like "Go on reading". For it to be able to close modal, add this attribute to the element code: ng-click="close()".

Changing backdrop color

Backdrop is located on your end, therefore all changes in CSS should be done in your CSS files as well. Here is the documentation on how to change the backdrop color.

About modal's position on a page

A modal is centered horizontally on a page (or takes 100% of its width, on narrow screens). Vertically, it's located 50px below the page top. This is resulted from years of QA. Our CSS should be universal and work on 100% websites, browsers, and devices. We strongly recommend not to break into it. What can be updated in a modal position is its top margin. Horizontal position and the code responsible for it should not be updated.
